Welcome to my Coding Lesson Plan!

February 26, 2016 by Sydney

Are you interested in learning how to code / program, but you have no technical experience? Or find the world of learning to code overwhelming (and sometimes expensive)? Have you tried to learn to code before and given up out of sheer frustration? Are you excited to learn and ready to become a programmer, but feel powerless to achieve your goal?


A year ago, I was in a similar place.


I had been working in Silicon Valley in a non-technical role for almost a year. All my friends (and my boyfriend) were software engineers. I saw how powerful knowing computer science / programming could be and knew I wanted to be in tech.


Yet, every time I tried to learn, I'd hit a wall. There were enough free resources in obvious places to teach me the basics of a few languages, but, after I had exhausted these, everything else I found was geared towards experts / those who already knew how to code professionally. I was unable to parse through these resources and my growth stalled.



I was willing to work hard to learn, but nothing I could find bridged me from those beginner stages to an experienced software engineer. Eventually, I signed up for a dev bootcamp (DevBootcamp). I had / am having a great experience in DBC, but know that its price (over 10k in tuition and lost earnings for the 10 weeks of onsite in the program) can be prohibitive for others.


Now that I've finally begun to crack this programming nut, I wanted to find a way to help those who don't have a bootcamp like I did to help them. So, I've decided to write an extensive lesson plan with links to only FREE materials that can take you from 0 to 100 when it comes to coding / software engineering.


NOTE:

  1. This guide is mainly for MAC users although most of the materials will also work for Windows
  2. I plan to update this guide periodically as I learn
  3. This is only ONE track for learning some programming concepts. Obviously, there are a lot of other ways to go. I am mostly mimicking the way I learned.
  4. This is a WORK IN PROGRESS ... feel free to send me feedback / notes (email me at srossmanreich@gmail.com or tweet at me)
